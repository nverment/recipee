# 1. CLI input + autocomplete for ingredients

### 👉 Use: `readline` (built-in, Unix/macOS)

or `prompt_toolkit` (cross-platform, more powerful)

---

## Option A — simple: `readline`

Good if you're on Linux/macOS.

### How it works:

* You define a **completer function**
* It suggests matches when the user presses TAB

### Basic idea:

```python
import readline

ingredients_list = ["chicken", "chickpeas", "yoghurt", "pita", "lentils"]

def completer(text, state):
    options = [i for i in ingredients_list if i.startswith(text)]
    return options[state] if state < len(options) else None

readline.set_completer(completer)
readline.parse_and_bind("tab: complete")
```

Then:

```python
input("Ingredient: ")
```

### Limitations:

* Only completes **one word at a time**
* Harder to handle lists like: `"chicken, yogh..."`

---

## Option B — better: `prompt_toolkit` ⭐ (recommended)

### Install:

```bash
pip install prompt_toolkit
```

### Why it's better:

* Handles **multi-word input**
* Better autocomplete UI
* Works on Windows too

### Basic usage:

```python
from prompt_toolkit import prompt
from prompt_toolkit.completion import WordCompleter

ingredients = ["chicken", "chickpeas", "yoghurt", "pita", "lentils"]

completer = WordCompleter(ingredients, ignore_case=True)

user_input = prompt("Ingredient: ", completer=completer)
```

### For multiple ingredients:

Let user type:

```
chicken, yogh<TAB>
```

Use:

```python
WordCompleter(ingredients, ignore_case=True, sentence=True)
```


  * Loop ingredients


# 9. Small design improvement

Avoid class-level mutable attributes like:

```python

ingredients = []
```

These are **shared across all instances** (bug-prone).

Use inside `__init__` instead.
